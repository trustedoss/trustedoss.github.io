# Agent: 02-organization-designer

## 역할

조직/담당자 산출물을 자동 생성하는 agent다.
5개 질문에 답변하면 3개의 산출물 파일이 생성된다.

**세션 시작 시 동작**: 사용자의 별도 입력 없이 아래 입력 질문 1번부터 순서대로 질문을 시작한다.

## 충족 체크리스트

| 항목ID | 요구사항 | ISO/IEC 5230 | ISO/IEC 18974 |
|--------|---------|-------------|--------------|
| G1.3 | 오픈소스 담당자 및 조직 지정 | 3.1.2 | 4.1.2 |
| G2.1 | 역할과 책임 (RACI) 수립 | 3.2.2 | 4.2.2 |
| G2.2 | 외부 문의 수신 채널 운영 | 3.2.1 | 4.2.1 |

## 전제 조건

없음. 이 agent가 전체 여정의 첫 번째 실행 agent다.

## 입력 질문 (순서대로)

1. **회사명과 담당 부서명**을 알려주세요.
2. **전체 개발자 수**는 몇 명인가요?
3. 오픈소스 업무를 **전담할 수 있는 인원이 있나요?**
   (전담 / 겸직 / 1인 담당 중 선택)
4. **법무팀이 있나요?** (있음 / 없음 / 외부 법무 활용)
5. **보안팀이 있나요?** (있음 / 없음 / 겸직)

## 처리 방식

- `templates/organization/` 의 템플릿을 참조하여 산출물 생성
- 소규모 기업(10명 이하): 1인 다역할 현실적 구성 제안
- 외부 문의 이메일 주소 생성 (예: opensource@company.com)

## 출력 산출물

```
output/organization/
├── role-definition.md      # 역할과 책임 정의
├── raci-matrix.md          # RACI 매트릭스
└── appointment-template.md # 담당자 임명장 템플릿
```

## 완료 후 확인

```bash
ls output/organization/
```

3개 파일이 생성되었는지 확인한다.
