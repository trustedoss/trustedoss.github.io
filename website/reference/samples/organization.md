# 조직 산출물 Best Practice

`organization-designer` agent가 생성하는 3개 산출물의 완성 예시입니다.
자신의 `output/organization/` 파일과 비교하여 빠진 항목을 확인하는 용도로 활용하세요.

> **레퍼런스 바로가기:** [조직 구성 챕터 가이드](/docs/02-organization)

---

## 스타트업 (개발자 10명 이하)

**프로필**: 개발자 8명, 1인 담당(CTO 겸직), 외부 법무 활용, 보안팀 없음

### role-definition.md

```markdown
# 오픈소스 역할 및 책임 정의

**회사명**: 클라우드샵(주)
**작성일**: 2026-01-10
**작성자**: 김민준 (CTO)

## 1. 오픈소스 프로그램 역할 목록

| 역할 | 담당자/부서 | 주요 책임 |
|------|-----------|---------|
| 오픈소스 담당자 (겸직) | 김민준 (CTO) | 정책 수립, 라이선스 검토, 외부 문의 대응 |
| 보안 담당자 (겸직) | 김민준 (CTO) | CVE 모니터링, 취약점 대응 |
| 법무 담당 | 법무법인 테크로 (외부 자문) | 라이선스 분쟁 자문, 법적 의무사항 확인 |
| 개발팀 대표 | 이수진 (시니어 개발자) | 프로세스 준수, SBOM 업데이트 |

## 2. 외부 문의 수신 채널

- **라이선스 컴플라이언스 문의**: opensource@cloudshop.io
- **보안 취약점 신고**: security@cloudshop.io
- **대응 담당자**: 김민준
- **응답 목표 시간**: 영업일 기준 3일 이내

## 3. 법률 자문 접근 방법

- **내부 법무팀**: 없음
- **외부 법무 활용**: 법무법인 테크로 (계약 시 또는 라이선스 분쟁 발생 시 활용)

## 4. 취약점 해결 전문성

- **담당 조직**: CTO 직접 대응
- **활용 가능한 외부 리소스**: KrCERT 취약점 정보 구독, GitHub Security Advisories
```

```markdown
## 역할별 담당자

|------|--------|--------|------|

## 예산 배분 현황

| 항목 | 현황 |
|------|------|
| 전담 인력 | 없음 (CTO 겸직) |
| 오픈소스 도구 예산 | 무료 도구만 활용 (syft, OSV API) |
| 법률 자문 예산 | 필요 시 외부 자문 (건별 청구) |
| 외부 교육 예산 | 없음 (무료 온라인 리소스 활용) |
```

## 중소기업 (개발자 10~100명)

**프로필**: 개발자 50명, OSPM 겸직 1명, 외부 법무 활용, 보안팀 겸직

### role-definition.md

```markdown
# 오픈소스 역할 및 책임 정의

**회사명**: (주)테크스타트
**작성일**: 2026-02-01
**작성자**: 박서영 (플랫폼팀 시니어 엔지니어, 오픈소스 담당)

## 1. 오픈소스 프로그램 역할 목록

| 역할 | 담당자/부서 | 주요 책임 |
|------|-----------|---------|
| 오픈소스 담당자 (겸직) | 박서영 (플랫폼팀) | 정책 수립, 라이선스 검토, 외부 문의 대응, 자체 인증 관리 |
| 보안 담당자 (겸직) | 정다은 (인프라팀) | CVE 스캔, 취약점 대응, SBOM 관리 |
| 법무 담당 | 법무법인 오픈IT (외부) | 라이선스 분쟁 자문, 계약서 검토 |
| 개발팀 대표 | 각 팀 리드 (5명) | 팀 내 프로세스 준수 감독, 신규 의존성 사전 검토 |

## 2. 외부 문의 수신 채널

- **라이선스 컴플라이언스 문의**: opensource@techstart.co.kr
- **보안 취약점 신고**: security@techstart.co.kr

- **응답 목표 시간**: 영업일 기준 2일 이내

## 3. 법률 자문 접근 방법

- **내부 법무팀**: 없음
- **외부 법무 활용**: 법무법인 오픈IT — 분기별 정기 자문 및 분쟁 발생 시 즉시 연락

## 4. 취약점 해결 전문성

- **담당 조직**: 인프라팀 (정다은 겸직)
- **활용 가능한 외부 리소스**: KrCERT 취약점 정보, GitHub Dependabot, OSV API
```

## 대기업 (개발자 100명 이상)

**프로필**: 개발자 300명, OSPO 전담팀 3명, 내부 법무팀·보안팀 있음

### role-definition.md

```markdown
# 오픈소스 역할 및 책임 정의

**회사명**: (주)엔터프라이즈솔루션
**작성일**: 2026-03-01
**작성자**: 최재원 (OSPO 팀장)

## 1. 오픈소스 프로그램 역할 목록

| 역할 | 담당자/부서 | 주요 책임 |
|------|-----------|---------|
| OSPO 팀장 (전담) | 최재원 (OSPO) | 전사 오픈소스 전략, 자체 인증 관리, 경영진 보고 |
| 오픈소스 컴플라이언스 담당 (전담) | 이하나 (OSPO) | 라이선스 검토, 정책 운영, 교육 |
| 오픈소스 보안 담당 (전담) | 김태현 (OSPO) | CVE 모니터링, 취약점 대응, SBOM 관리 |
| 법무팀 담당 | 권지수 (법무팀) | 오픈소스 관련 법적 자문, 계약서 검토 |
| 보안팀 담당 | 오민철 (보안팀) | 전사 보안 정책 연계, CVE 대응 협력 |
| 개발팀 대표 | 각 유닛 리드 (15명) | 현장 프로세스 준수, 신규 의존성 사전 검토 |

## 2. 외부 문의 수신 채널

- **라이선스 컴플라이언스 문의**: opensource@enterprise-solutions.com
- **보안 취약점 신고**: security@enterprise-solutions.com
- **대응 담당자**: 이하나 (라이선스), 김태현 (보안)
- **응답 목표 시간**: 영업일 기준 1일 이내 (초기 응답), 5일 이내 (최종 처리)

## 3. 법률 자문 접근 방법

- **내부 법무팀**: 있음 (권지수 변호사 전담)
- **외부 법무 보완**: 대형 분쟁 시 외부 법무법인 추가 자문

## 4. 취약점 해결 전문성

- **담당 조직**: OSPO 보안 담당(김태현) + 보안팀(오민철) 협업
- **활용 외부 리소스**: KrCERT, NVD, GitHub Security Advisories, Dependency Track 운영
```
