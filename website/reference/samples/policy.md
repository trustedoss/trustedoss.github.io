# 정책 산출물 Best Practice

`policy-generator` agent가 생성하는 2개 산출물의 완성 예시입니다.
배포 방식(SaaS / 앱스토어 / 임베디드)에 따라 라이선스 정책이 달라지므로,
자신의 상황에 맞는 프로필을 참고하세요.

> **레퍼런스 바로가기:** [오픈소스 정책 수립 챕터 가이드](/docs/03-policy)

## 스타트업 (SaaS, Python + JavaScript)

**프로필**: 8명, SaaS 서비스, 오픈소스 기여 계획 없음, 외부 납품 없음

### oss-policy.md (주요 섹션)

```markdown
# 오픈소스 정책

**회사명**: 클라우드샵(주)
**버전**: 1.0
**시행일**: 2026-01-15
**담당자**: 김민준 (CTO)

## 1. 목적 및 적용 범위

이 정책은 클라우드샵(주) 전 임직원이 오픈소스 소프트웨어를 사용·기여·배포할 때 적용된다.
SaaS 서비스 개발 및 운영에 사용되는 모든 오픈소스 컴포넌트를 대상으로 한다.

## 2. 오픈소스 사용 원칙

- Permissive 라이선스(MIT, Apache-2.0, BSD)는 별도 승인 없이 사용 가능하다.
- Copyleft 라이선스(GPL, LGPL 등)는 담당자 검토 후 사용한다.
  - SaaS 서비스 특성상 GPL은 서버 내부 사용으로 배포 의무가 발생하지 않으나, AGPL은 소스 공개 의무가 있어 사용 전 반드시 검토한다.
- 라이선스가 명시되지 않은 패키지는 사용하지 않는다.

## 3. 라이선스 의무 이행 절차

1. 신규 패키지 도입 시 license-allowlist.md 확인
2. 목록에 없는 라이선스는 담당자(opensource@cloudshop.io)에게 검토 요청
3. 연 1회 전체 의존성 라이선스 점검 실시

## 4. 보안 취약점 대응 절차

1. GitHub Dependabot 경보 즉시 확인
2. Critical/High CVE: 발견 후 7일 이내 패치 적용
3. Medium CVE: 다음 정기 배포 시 포함

## 5. 정책 검토 주기

연 1회 (매년 1월) 검토 및 개정. 변경 시 전 임직원에게 이메일로 공지한다.
```

### license-allowlist.md (주요 섹션)

```markdown
# 오픈소스 허용 라이선스 목록

**회사명**: 클라우드샵(주) | **배포 방식**: SaaS

## 허용 라이선스 (자동 승인)

| 라이선스 | 분류 | SaaS 내부 사용 | SaaS 배포 | 비고 |
|---------|------|:-----------:|:-------:|------|
| MIT | Permissive | ✓ | ✓ | 고지 의무 있음 |
| Apache-2.0 | Permissive | ✓ | ✓ | 고지 의무 있음 |
| BSD-2-Clause | Permissive | ✓ | ✓ | |
| BSD-3-Clause | Permissive | ✓ | ✓ | |
| ISC | Permissive | ✓ | ✓ | |
| CC0-1.0 | Public Domain | ✓ | ✓ | |
| GPL-2.0, GPL-3.0 | Strong Copyleft | ✓ | ✓ | SaaS는 배포에 해당 안 됨 |
| LGPL-2.1, LGPL-3.0 | Weak Copyleft | ✓ | ✓ | 동적 링킹 조건 |

## 검토 필요 라이선스 (담당자 승인 후 사용)

| 라이선스 | 이유 |
|---------|------|
| AGPL-3.0 | 네트워크 서비스 제공 시 소스 공개 의무 — SaaS에 직접 영향 |
| EUPL | 국내 적용 불확실성 |
| CDDL | 특허 조항 검토 필요 |

## 사용 금지 라이선스

| 라이선스 | 이유 |
|---------|------|
| UNLICENSED | 법적 보호 없음 |
| 상용 라이선스 오픈소스 오표기 | 계약 위반 가능성 |
```

## 중소기업 (SaaS + 앱스토어 배포)

**프로필**: 50명, SaaS + 모바일 앱 배포, 외부 고객에 API 납품

### oss-policy.md (주요 섹션)

```markdown
# 오픈소스 정책

**회사명**: (주)테크스타트
**버전**: 1.0
**시행일**: 2026-02-15
**담당자**: 박서영 (플랫폼팀 오픈소스 담당)

## 1. 목적 및 적용 범위

SaaS 서비스, 모바일 앱(iOS/Android), 외부 납품 API를 포함한 전 제품에 적용된다.
배포 방식별로 라이선스 의무 기준이 다르므로 이를 반드시 구분하여 준수한다.

## 2. 배포 방식별 라이선스 기준

| 배포 방식 | GPL 허용 여부 | AGPL 허용 여부 | 주의 사항 |
|---------|:-----------:|:------------:|---------|
| SaaS 내부 사용 | ✓ | ✗ (검토 필요) | AGPL은 네트워크 배포 조항 확인 |
| 모바일 앱 배포 | ✗ (검토 필요) | ✗ | Copyleft 오염 위험 |
| 외부 API 납품 | ✗ (검토 필요) | ✗ | 고객사 요구사항 반영 |

## 3. 납품 시 요구사항

외부 고객에게 소프트웨어를 납품할 경우:
- 요청 시 SBOM(CycloneDX 포맷) 제공 의무
- 라이선스 고지문(NOTICE 파일) 동봉
- 계약서에 오픈소스 관련 조항 포함 여부 확인

## 4. 정책 검토 주기

연 1회 (매년 1월) 검토. 납품처 요구 변경 시 즉시 업데이트.
```

## 대기업 (멀티 배포: SaaS + 임베디드 + 납품)

**프로필**: 300명+, SaaS/모바일/임베디드 복합 배포, 대형 고객사 납품

### oss-policy.md (주요 섹션)

```markdown
# 오픈소스 정책

**회사명**: (주)엔터프라이즈솔루션
**버전**: 2.1
**시행일**: 2026-03-01
**담당자**: 이하나 (OSPO 컴플라이언스 담당)
**승인자**: 최재원 (OSPO 팀장)

## 1. 목적 및 적용 범위

이 정책은 SaaS 플랫폼, 모바일 앱, 임베디드 소프트웨어, 외부 납품 제품에 모두 적용된다.
배포 유형별 의무사항이 상이하므로 개발팀은 반드시 해당 배포 유형을 확인 후 적용한다.

## 2. 배포 유형별 라이선스 기준

| 배포 유형 | Permissive | Weak Copyleft | Strong Copyleft | AGPL |
|---------|:----------:|:-------------:|:---------------:|:----:|
| SaaS 내부 | ✓ | ✓ | ✓ | ✗ |
| 앱스토어 배포 | ✓ | △ (동적 링킹) | ✗ | ✗ |
| 임베디드/펌웨어 | ✓ | △ (별도 검토) | ✗ | ✗ |
| 대형 고객 납품 | ✓ | △ (계약 확인) | ✗ | ✗ |

## 3. OSPO 검토 의뢰 기준

아래 경우 개발팀은 OSPO에 검토 의뢰 후 사용한다:
- Copyleft 라이선스를 임베디드/배포 제품에 사용 시
- 신규 오픈소스 라이선스(허용 목록 외) 사용 시
- 납품 계약서에 오픈소스 관련 특별 조항 포함 시

## 4. 오픈소스 기여 정책

회사 업무로 오픈소스 프로젝트에 코드를 기여할 경우:
1. OSPO에 기여 계획 사전 신고
2. 회사 IP 포함 여부 법무팀 확인
3. CLA(기여자 라이선스 계약) 서명 전 법무 검토
```
