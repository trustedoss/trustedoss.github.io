## 1. 자체 인증 등록 절차

### 1단계 — 사전 확인

등록 전 아래 항목을 확인한다:

- [ ] `output/conformance/gap-analysis.md` 완료 — 미충족 항목 없거나 해소 계획 있음
- [ ] `output/conformance/declaration-draft.md` 최종 검토 완료
- [ ] 선언 범위(적용 소프트웨어/제품)가 명확히 정의됨
- [ ] 승인 권한자(경영진 또는 담당 임원)의 사전 동의 확보

### 2단계 — OpenChain 웹사이트 접속

https://www.openchainproject.org/conformance

### 3단계 — 표준 선택 및 양식 작성

| 인증할 표준 | 선택 |
|-----------|------|
| ISO/IEC 5230 (라이선스 컴플라이언스) | ☐ |
| ISO/IEC 18974 (보안 보증) | ☐ |
| 두 표준 모두 | ☐ |

양식 작성 시 `output/conformance/declaration-draft.md` 내용을 참고한다.

### 4단계 — 체크리스트 항목 확인 및 제출

OpenChain 온라인 양식의 각 항목에 체크하고 제출한다.
제출 완료 시 확인 이메일이 수신된다.

### 5단계 — 인증 완료 확인

등록 완료 후:
- OpenChain 공식 인정 기업 목록에 등재됨
- OpenChain 로고 사용 가능 (사용 조건 확인 필요)

## 3. 재선언 준비 절차 (18개월 후)

재선언 시기가 되면 아래 순서로 준비한다:

1. `agents/07-conformance-preparer`를 재실행하여 갭 분析 업데이트
2. 변경된 정책·프로세스 반영 여부 확인
3. 지난 18개월간의 취약점 대응 이력 및 교육 이력 확인
4. `output/conformance/declaration-draft.md` 업데이트 (날짜, 범위 변경 여부 등)
5. OpenChain 웹사이트에서 재선언 등록
