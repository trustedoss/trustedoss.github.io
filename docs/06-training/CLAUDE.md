# 챕터 06 — 교육 체계

## 현재 위치: 6단계 - 교육 체계 (6/7단계)

## 이 챕터의 목표

직군별 교육 커리큘럼을 수립하고, 이수 추적 체계를 만들며,
무료로 활용할 수 있는 교육 리소스를 정리한다.

두 표준 모두 담당자와 관련 구성원이 교육을 이수했다는 증빙을 요구한다.
이 챕터에서는 교육 이수 기록을 관리할 수 있는 체계를 구축한다.

## 충족되는 체크리스트 항목

| 항목ID | 요구사항 | ISO/IEC 5230 | ISO/IEC 18974 |
|--------|---------|-------------|--------------|
| G1.4 | 교육 프로그램 수립 | 3.1.2 | 3.1.2 |
| G2.3 | 인식 제고 프로그램 운영 | 3.1.3 | 3.1.3 |

> 이 단계는 ISO/IEC 5230 3.1.2, 3.1.3 및 ISO/IEC 18974 동일 항목 요구사항을 충족합니다.

## agent 실행 안내

```bash
cd agents/06-training-manager
claude
```

agent가 아래 질문을 순서대로 한다:
1. 직군별 인원 (개발자 N명 / 관리자 N명 / 운영 N명)
2. 선호하는 교육 형태 (온라인 자율 / 오프라인 집합 / 혼합)
3. 교육 이수 증빙 목적 (내부 기록용 / 감사 대비 / 인증 제출용)

## 무료 교육 리소스

| 리소스 | 내용 | 소요시간 |
|--------|------|---------|
| OpenChain 교육 자료 | ISO/IEC 5230 및 18974 공식 교육 | 자율 |
| Linux Foundation (LFC193) | 오픈소스 라이선스 컴플라이언스 | 3시간 |
| Linux Foundation (LFD106x) | 오픈소스 개발 기초 | 자율 |
| SPDX 교육 | SBOM 및 SPDX 표준 | 1시간 |

agent가 생성하는 `output/training/resources.md` 에 링크와 함께 정리된다.

## 자주 발생하는 문제

**Q: 교육 이수 증빙을 어떻게 하나요?**
A: 가장 간단한 방법은 이메일 확인서 또는 서명된 completion-tracker 스프레드시트다.
외부 LMS를 사용한다면 수료증을 출력하여 보관한다.

**Q: 신규 입사자 교육은 어떻게 하나요?**
A: curriculum.md 에 온보딩 교육 항목을 포함한다. 입사 후 30일 이내 이수를 권장한다.
