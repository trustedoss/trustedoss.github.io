# 오픈소스 담당자 역할 정의서

작성일: 2026-XX-XX
버전: 1.0
작성자: [오픈소스 프로그램 매니저]

---

## 1. 오픈소스 프로그램 매니저 (OSPM)

**역할**: 오픈소스 프로그램 전반의 기획, 운영, 개선을 총괄한다.

**주요 책임**:
- 오픈소스 정책 수립 및 갱신
- 오픈소스 사용 승인 최종 결정
- 외부 오픈소스 커뮤니티 및 기관과의 협력
- ISO/IEC 5230 & 18974 자체 인증 관리

**자격 요건**:
- 오픈소스 라이선스 기초 지식
- 소프트웨어 개발 또는 법무 배경 (권장)

---

## 2. 오픈소스 보안 담당자

**역할**: 오픈소스 구성 요소의 취약점을 식별하고 대응한다.

**주요 책임**:
- SBOM 기반 CVE 취약점 정기 스캔
- 취약점 위험 점수(CVSS) 평가 및 우선순위 결정
- 패치 적용 또는 대체 컴포넌트 선정
- 취약점 대응 이력 관리

**자격 요건**:
- 보안 취약점 기초 지식 (CVSS, NVD 활용)
- SBOM 도구 사용 경험 (권장)

---

## 3. 오픈소스 법무 검토자

**역할**: 라이선스 의무사항의 법적 해석과 컴플라이언스 이행을 지원한다.

**주요 책임**:
- 라이선스 분쟁 위험 검토
- Copyleft 라이선스 의무 이행 검증
- 오픈소스 기여 IP 검토
- 외부 문의(라이선스 의무 이행 요청) 대응

**자격 요건**:
- IP/저작권법 기초 지식
- 오픈소스 라이선스 유형별 차이 이해

---

## 관련 표준 요구사항

> 이 문서는 ISO/IEC 5230 3.1.2 및 ISO/IEC 18974 3.1.2 요구사항을 충족합니다.
