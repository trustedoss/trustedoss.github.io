# 오픈소스 사용 승인 절차

버전: 1.0
갱신일: 2026-XX-XX

---

## 1. 개요

신규 오픈소스 컴포넌트를 프로젝트에 도입할 때 따르는 승인 절차를 정의한다.

---

## 2. 승인 절차 흐름

```
개발자 도입 요청
       ↓
라이선스 확인 (허용 목록 대조)
       ↓
[허용 라이선스?]
   ✅ YES → 자동 승인 (허용 목록 기재 라이선스)
   ⚠️ 조건부 → OSPM 검토 → 승인/거절
   ❌ 금지 → 법무 검토 → 예외 승인 or 거절
       ↓
SBOM 업데이트
       ↓
CVE 스캔 실행
       ↓
승인 완료 → 도입
```

---

## 3. 승인 요청 양식

| 항목 | 내용 |
|------|------|
| 컴포넌트명 | (예: `lodash`) |
| 버전 | (예: `4.17.21`) |
| 라이선스 | (예: `MIT`) |
| 사용 목적 | (예: 유틸리티 함수 모음) |
| 배포 방식 | 내부 배포 / 외부 배포 / SaaS |
| 요청자 | |
| 요청일 | |

---

## 4. 처리 기준

| 라이선스 분류 | 처리 방식 | 목표 처리 시간 |
|-------------|---------|-------------|
| 허용 라이선스 | 자동 승인 | 즉시 |
| 조건부 라이선스 | OSPM 검토 | 3 영업일 |
| 금지 라이선스 | 법무 검토 | 7 영업일 |
